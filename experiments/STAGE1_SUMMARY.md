## 発見

### Claudeの強み
- コード品質（動くHTML/CSSが即出る）
- レイアウトの工夫（横スクロール＋縦リスト＋バナー等）
- 情報密度（距離、時間、空席状況等の実用情報）
- 日本語UIの自然さ

### Claudeの弱点
1. **画像** — 絵文字に逃げる傾向。Unsplash使用で改善可能だが、適切な写真選びが課題（温泉にリゾートプール画像）
2. **暗いデザインへの偏り** — デフォルトがシック・ダーク系。用途に応じた色選びができていなかった
3. **ユーザー理解の浅さ** — 最大の敗因。60代向けで電話予約ボタン・ヘルプセクションを入れられなかった。「利用者の行動パターン」まで考えが及ばない
4. **機能設計の漏れ** — 初回はカテゴリフィルター・星評価を入れ忘れた（2回目以降で改善）

### Figmaの強み
- リアルな写真の配置
- ユーザー理解に基づいたUI/UX判断（電話予約、ヘルプ等）
- レスポンシブ対応
- React + Tailwindのコード出力

### 前提の崩壊
当初「Figmaはデザインだけ、Claudeはコードも出せるから有利」と考えていたが、FigmaもReact + Tailwindコードを出力できることが判明。コード出力の優位性は薄れた。

## 次のステップへの示唆

比較ポイントを再整理する必要がある：
- ~~コード出力の有無~~ → 両方できる
- **UI/UXの設計判断の質** → Figmaが上（特にターゲット特化時）
- **ユーザー理解の深さ** → Claudeの最大の改善ポイント
- **デザインの方向性の適切さ** → 用途に応じた判断力が必要
